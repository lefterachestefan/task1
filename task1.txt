Global Technical Rules

SQLite is not allowed for this challenge.

Your API must query data from the database path throughout the challenge.

Objective
Build core API behavior for:

Filtered listing search (GET /listings)
Single listing details (GET /listings/{listing_id})

Use the 1k SQL dump and load it into MySQL or PostgreSQL.
Data Setup
Download the 1k SQL dump from: https://drive.google.com/file/d/1jT_isHO5P08hO38_DTeTsdO__FIbmfD-/view?usp=sharing
Create a local MySQL or PostgreSQL database.
Import the SQL dump.
Configure your app to read from that database.
Required Endpoints
GET /health
Returns {"status":"ok"}.
GET /listings
Return listing summaries with support for these query parameters:

min_rooms, max_rooms
min_price, max_price
listing_type (sale or rent)
min_area, max_area
min_floor, max_floor
tags (comma-separated, AND semantics)
min_lat, max_lat, min_lon, max_lon (inclusive bbox)
limit (1..500, default 100)

Response items must include:

id, rooms, area_sqm, price, listing_type, tags, lat, lon, floor

Do not include title or description in search responses.

Sort results by ID.
GET /listings/{listing_id}
Return full listing details for a single listing id, including:

id, title, description, rooms, area_sqm, price, listing_type, tags, lat, lon, floor
